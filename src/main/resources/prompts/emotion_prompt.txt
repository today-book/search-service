아래 문장에 대해 다음 두 값을 정확한 JSON 형식으로만 반환하세요.

1) emotion:
   - 문장의 감정을 ["JOY", "SAD", "FEAR", "ANGER", "HEALING", "MOTIVATION", "NEUTRAL"] 중 하나로 선택
   - 반드시 목록 중 하나만 사용

2) query:
   - 사용자의 성격(personality)과 요구사항(requirements)을 핵심 요소로 삼아 임베딩 검색에 최적화된 한 문장 생성
   - 입력 문장에 포함된 성격, 취향, 학습 목적, 주제(예: 경제학)를 반드시 반영
   - 감정(emotion)과 자연스럽게 결합하되, 감정이 핵심 의미를 왜곡하면 안 됨
   - 원본 의미(질의 의도 + 성격 + 탐구 주제)를 우선적으로 유지·강조
   - 한 문장만 생성, 불필요한 말 제거
   - 문장의 마지막 단어는 반드시 "책" 또는 "책." 으로 끝나야 함
   - 모호한 표현 금지 (예: "~같은", "~풍의", "~느낌의")

입력 문장: {query}

⚠️ 출력 규칙 (매우 중요)
- 반드시 JSON ONLY 출력 (설명·부가 문장·코드블록 금지)
- 출력 JSON은 다음 두 key만 포함: "emotion", "query"
- key 순서 유지: emotion → query
- null, 빈 값, 추가 필드 절대 금지
- 예시 형식:
  \{"emotion": "NEUTRAL", "query": "호기심 많은 성향으로 학습의 본질을 탐구하며 경제학적 통찰을 얻을 수 있는 책"\}
